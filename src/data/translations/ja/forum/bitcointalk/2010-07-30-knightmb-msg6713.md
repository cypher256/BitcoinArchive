> [Quote from: satoshi on July 30, 2010, 07:53:06 PM](#msg6711)
> なぜそんなに苦労しているのか理解できない。build-unix.txtの手順に従っただけだ。Boost 1.37用にいくつか小さな修正を加えたので、次回SVNを更新する時に反映する。以下に記す：
> 
> Dependencies
> ------------
> sudo apt-get install build-essential
> sudo apt-get install libgtk2.0-dev
> sudo apt-get install libssl-dev
> sudo apt-get install libdb4.7-dev
> sudo apt-get install libdb4.7++-dev
> sudo apt-get install libboost-all-dev (or libboost1.37-dev)

ハイライトした部分が引っかかったところだ。あのコマンドではすべてのlibboostパッケージをインストールできないし（*を使ってずるしようとしたけどダメだった）、問題の一部は、libboostに関する*すべて*をインストールするんだと文字通りに受け取りすぎたことだ（笑）。

実際に必要だったのはlibboost1.37-devパッケージだけで、それでコンパイルエラーがすべて消えた。自前でwxWidgetsをコンパイルしたり、Boost 1.4をコンパイルしたりする以外は問題なく動いた。

つまり、最後のコマンドは単純に
**sudo apt-get install libboost1.37-dev**
とすべきだ。

余談だが、Ubuntu 64ビットシステムでコンパイルしたので、完成したプログラムは64ビット対応になった。いくつかの64ビットシステムでテスト中だ。
