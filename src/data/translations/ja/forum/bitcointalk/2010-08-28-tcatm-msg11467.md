ソースコードを見直した。さらに最適化するアイデアがいくつかあったのだが、4wayが部分的に壊れていることに気づいた：

main.cppから：
Code:                for (int j = 0; j < NPAR; j++) 
                {    
                    if (thash[7][j] == 0)
                    {                        
                        for (int i = 0; i < sizeof(hash)/4; i++) 
                          ((unsigned int*)&hash)[i] = thash[i][j];
                        pblock->nNonce = ByteReverse(tmp.block.nNonce + j);
                    }    
                }    

このコードは、thash[7] == 0のハッシュが複数あっても、32個のハッシュのうち1つ（thash[7] == 0の最後のもの）しか処理しない。
