[Quote from: BitcoinFX on February 01, 2010, 10:08:54 PM](#msg192)
> 「Your application (using socks5 on port xxxx) is giving Tor only an IP address. Applications that do DNS resolves themselves may leak information. Consider
> using Socks4A (e.g. via polipo or socat) instead.」

BitcoinはIPアドレスを使っていて、ホスト名は使っていないので、DNSは不要だ。TorはbitcoinがTorの内部DNSを通さずにIPに接続しようとしているため、通常のDNSを使っていると判断してしまう。

[Quote from: BitcoinFX on February 01, 2010, 10:08:54 PM](#msg192)
> しかし、ポート8333（Bitcoinの「デフォルト」として想定通り）と6667（間違っていなければIRCポートだ！？）について、時折警告が出る。

Bitcoinはポート8333を使っている。Torのポート9050を通してリレーしているとしてもだ。Tongue
6667はIRCだ。Bitcoinはノードリストの配布にIRCサーバーを使っている。（Bitcoinを動かしている別のコンピュータのIPを知っていれば、-connectオプションを指定してノードリストの使用を回避できる。）

[Quote from: BitcoinFX on February 01, 2010, 10:08:54 PM](#msg192)
> しかし、Bitcoinはブロックの漏れがないか確認するために、すべてのノードに接続を試みなければ*ならない*！

いや、最長のブロックチェーンのコピーを持っている限り、たった1つのノードに接続されていれば十分だ。
