いくつかのトピック（https://www.bitcoin.org/smf/index.php?topic=12.0、https://www.bitcoin.org/smf/index.php?topic=19.0）で読んだように、Bitcoin のより広い普及のためには決済サーバーが必要だ。

Bitcoin トランザクションを処理できる PHP Web アプリケーションを作りたい。LAMP システムは稼働しているが、今まで Bitcoin は Windows でしか使っていなかった。コマンドラインインターフェースを備えた Linux 版 Bitcoin のインストールを手伝ってくれる人はいるだろうか？

サーバーは Ubuntu 9.10 32-bit で動いている。それとも別のディストリビューションを使うべきだろうか？

私のアイデアはシンプルだ：

- 訪問者が何かを購入したい
- 決済ページに支払うべき Bitcoin の金額、注文 ID、サーバーの IP アドレスが表示される（メッセージは IP アドレストランザクションでしか送れないため）
- 訪問者が Bitcoin アプリケーションで支払いを行い、支払いのメッセージに注文 ID を記載する
- サーバーが支払いを受信し、注文 ID と金額を確認する
- 支払いが確認され、訪問者にダウンロードリンク/購入確認などが送られる

Bitcoin 0.2 でこれは可能だろうか？
