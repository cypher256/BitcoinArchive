どこまで凝りたいか次第だな。

ブール値を超えて、必要な機能ごとにフラグを追加していくこともできる。そうすれば、機能が増えるたびに引数にブール値を山積みしていくのではなく、新しい機能に対して開かれた形にできる。

sendtoaddress <bitcoinaddress> <amount> [comment] [comment-to]
-tx_id : 受け取り先bitcoinアドレスのtx IDを返す
-dance : ダンスのハイスコアを返す
-moon : 現在の月相を返す

つまり、後で多くの機能が追加されたときも、
sendtoaddress <bitcoinaddress> <amount> [comment] [comment-to] true false true true false
の代わりに、単純に
sendtoaddress -tx-_id <bitcoinaddress> <amount> [comment] [comment-to]
と書けるようになる。
