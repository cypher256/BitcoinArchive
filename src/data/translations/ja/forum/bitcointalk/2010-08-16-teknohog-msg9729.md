Core 2 Duo T7200では、デフォルトのコードで約1.8 Mhash/sが出るが、4wayは1.0 Mhash/sと遅い。L2キャッシュは4MBなので、どこかで示唆されたようなキャッシュサイズの問題ではないだろう。

残念ながら、（svnからの）コードはSSEイントリンシクスがハードコードされているため、ARMでコンパイルできなくなった。makefileから-msse2と-DFOURWAYSSE2フラグを削除したが、まだこのようなエラーが出る

Code:sha256.cpp:8:23: error: xmmintrin.h: No such file or directory
sha256.cpp:34: error: '__m128i' does not name a type

しかし、これは簡単に修正できることを期待している。
